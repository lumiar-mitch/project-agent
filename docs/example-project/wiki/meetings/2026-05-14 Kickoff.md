# Kickoff — 2026-05-14

Synthesis of the project kickoff. Verbatim minutes are in the
[source](../../raw/sources/meetings/2026-05-14%20Kickoff.md); this page holds what it means and
where it connects.

## What was settled

- **Platform decision made.** The sponsor confirmed the new site will be built on the Northbank CMS
  within the council cloud tenancy — logged as [[DEC-001]] with the three options considered. This
  clears the biggest open question from the initiation phase and lets design start.
- **Scope boundary held on the finance integration.** The steering committee confirmed the rates/
  finance back-end integration stays out of this phase; forms will hand off by email and reference
  number for now. Worth watching as a future dependency, not a current one.

## What surfaced

- **Accessibility is the early risk.** The current site's WCAG failures are part of the reason for
  the project, and the group recognised the new site could repeat the pattern if accessibility is
  left to a late audit. Raised as [[RSK-001]] with a named owner (Leah Consadine, external advisor)
  rather than "the team", and a review date of 2026-06-11. Primary mitigation is [[ACT-001]] —
  bake WCAG acceptance criteria into the definition of done so it is checked every sprint.
- **Content migration looks larger than estimated.** Tom Bright flagged that the ~400 "priority"
  pages contain a lot of duplicated and out-of-date content that needs rationalising before
  migration, not just lifting across. Not yet a formal register item — captured as an open question
  for the content workstream to size before the next steering meeting.

## Coach's note

The risk was logged with a named owner and a live review date, and framed as cause → risk → effect —
good practice per `standards/risk-quality`. The content-migration concern is a genuine emerging risk;
it has been left as an open question rather than logged prematurely, because there is no owner or
sizing yet. Flag to size it before the next steering meeting so it does not drift.
